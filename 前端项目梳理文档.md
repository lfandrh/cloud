# 微服务系统设计方案

## 第一章：前端项目架构分析与适配要点

### 1.1 项目概述

**SoybeanAdmin** 是一个基于 Vue3、Vite7、TypeScript、Pinia 和 UnoCSS 的现代化后台管理模板，采用 pnpm monorepo 架构，具有清新优雅的界面设计和强大的功能特性。

### 1.2 技术栈分析

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue | 3.5.26 | 前端框架 |
| Vite | 7.3.0 | 构建工具 |
| TypeScript | 5.9.3 | 类型系统 |
| Pinia | 3.0.4 | 状态管理 |
| Vue Router | 4.6.4 | 路由管理 |
| NaiveUI | 2.43.2 | UI组件库 |
| UnoCSS | 66.5.10 | CSS原子化框架 |
| Alova | 3.4.1 | 现代HTTP请求库 |
| Axios | 1.13.2 | 传统HTTP请求库 |


### 1.4 HTTP客户端配置

#### 1.4.1 双请求库支持

项目同时支持 **Alova** 和 **Axios** 两种HTTP客户端，开发者可根据需要选择：

| 特性 | Axios (@sa/axios) | Alova (@sa/alova) |
|------|-------------------|-------------------|
| 文件位置 | `src/service/` | `src/service-alova/` |
| 特点 | 传统拦截器模式 | 响应式、现代化 |
| Token刷新 | onBackendFail拦截器 | tokenRefresher配置 |
| Mock支持 | 需额外配置 | 内置mockAdapter |

#### 1.4.2 核心配置参数

```typescript
// 请求超时时间
const timeout = 10000;  // 10秒

// 请求头配置
headers: {
  'Content-Type': 'application/json',
  'X-Request-Id': nanoid()  // 每个请求唯一ID
}

// Token获取方式
function getAuthorization() {
  const token = localStorage.getItem('token');
  return token ? `Bearer ${token}` : null;
}
```

### 1.5 路由管理模式

#### 1.5.1 路由模式配置

```env
# .env 配置
VITE_AUTH_ROUTE_MODE=static      # static | dynamic
VITE_ROUTE_HOME=home             # 默认首页
VITE_ROUTER_HISTORY_MODE=history # hash | history | memory
VITE_STATIC_SUPER_ROLE=R_SUPER   # 静态超管角色
```

#### 1.5.2 静态路由模式

- 路由在前端代码中定义 (`src/router/elegant/routes.ts`)
- 基于文件系统自动生成路由配置
- 使用Elegant Router插件自动处理导入和类型

**路由数据结构：**
```typescript
interface ElegantConstRoute {
  name: string;           // 路由名称
  path: string;           // 路由路径
  component: string;      // 组件路径
  meta: {
    title: string;        // 标题
    i18nKey: string;      // 国际化key
    icon?: string;        // 图标
    order?: number;       // 排序
    roles?: string[];     // 角色权限
    constant?: boolean;   // 是否为常量路由
    hideInMenu?: boolean; // 是否在菜单中隐藏
    keepAlive?: boolean;  // 是否缓存
  };
  children?: ElegantConstRoute[];
}
```

#### 1.5.3 动态路由模式

- 从后端API获取用户路由 (`GET /route/getUserRoutes`)
- 后端返回路由配置后，前端动态添加到Vue Router

**动态路由API响应格式：**
```typescript
interface UserRoute {
  routes: MenuRoute[];     // 用户有权访问的路由列表
  home: string;            // 用户首页路由key
}

interface MenuRoute extends ElegantConstRoute {
  id: string;              // 路由ID
}
```

### 1.6 权限认证机制

#### 1.6.1 认证流程

```
登录页面
   ↓ 输入用户名密码
POST /auth/login
   ↓ 返回 { token, refreshToken }
存储到 localStorage
   ↓
GET /auth/getUserInfo
   ↓ 返回 { userId, userName, roles[], buttons[] }
存储用户信息和权限
   ↓
初始化权限路由
   ↓
跳转到首页
```

#### 1.6.2 Token管理机制

```typescript
// Token存储位置
localStorage.setItem('token', loginToken.token);
localStorage.setItem('refreshToken', loginToken.refreshToken);

// Token刷新机制
async function handleRefreshToken() {
  const rToken = localStorage.getItem('refreshToken');
  const { data } = await fetchRefreshToken(rToken);
  localStorage.setItem('token', data.token);
  localStorage.setItem('refreshToken', data.refreshToken);
}
```

#### 1.6.3 状态码处理机制

```env
# .env 状态码配置
VITE_SERVICE_SUCCESS_CODE=0000           # 成功码
VITE_SERVICE_LOGOUT_CODES=8888,8889      # 直接登出码
VITE_SERVICE_MODAL_LOGOUT_CODES=7777,7778 # 弹窗确认登出码
VITE_SERVICE_EXPIRED_TOKEN_CODES=9999,9998,3333 # Token过期码（触发刷新）
```

**状态码处理逻辑：**
| 状态码类型 | 处理方式 |
|-----------|---------|
| 0000 | 请求成功，返回data |
| logoutCodes | 直接退出登录，跳转登录页 |
| modalLogoutCodes | 弹窗提示，确认后退出 |
| expiredTokenCodes | 自动刷新token，重试请求 |

#### 1.6.4 角色权限控制(RBAC)

```typescript
// 用户信息结构
interface UserInfo {
  userId: string;
  userName: string;
  roles: string[];      // 角色列表，如 ['R_SUPER', 'R_ADMIN']
  buttons: string[];    // 按钮权限列表

// 路由权限检查
const hasAuth = authStore.isStaticSuper || 
                !routeRoles.length || 
                authStore.userInfo.roles.some(role => routeRoles.includes(role));
```

### 1.7 接口约定规范

#### 1.7.1 统一响应格式

```typescript
// 标准响应结构
interface Response<T = unknown> {
  code: string;    // 状态码 (0000表示成功)
  msg: string;     // 消息
  data: T;         // 数据
}

// 分页数据结构
interface PaginatingQueryRecord<T> {
  current: number;   // 当前页
  size: number;      // 每页大小
  total: number;     // 总记录数
  records: T[];      // 数据列表
}
```

#### 1.7.2 请求头规范

| Header | 说明 | 必需 |
|--------|------|------|
| Authorization | Bearer Token | 登录后必需 |
| X-Request-Id | 请求唯一标识 | 是 |
| Content-Type | application/json | 是 |

#### 1.7.3 分页参数约定

```typescript
interface PaginatingCommonParams {
  current: number;  // 当前页码 (从1开始)
  size: number;     // 每页条数
}
```

#### 1.7.4 状态枚举定义

```typescript
// 启用状态

// 菜单类型
```


### 1.10 前端架构特点总结

1. **Monorepo结构**：使用pnpm workspace管理多个子包，便于复用和维护
2. **双请求库支持**：同时支持Alova和Axios，开发者可选择熟悉的方案
3. **自动化路由**：基于Elegant Router，根据文件系统自动生成路由配置
4. **灵活权限**：支持前端静态路由和后端动态路由两种模式
5. **完整类型支持**：TypeScript严格类型检查，包含完整的API类型定义
6. **现代化状态管理**：使用Pinia进行状态管理，支持模块化
7. **国际化支持**：内置i18n方案，支持多语言切换

### 1.11 后端适配关键要点

基于以上分析，后端微服务设计需要关注以下关键要点：

1. **统一响应格式**：所有API必须返回 `{ code, msg, data }` 格式，成功code为"0000"
2. **Token机制**：支持双Token机制（accessToken + refreshToken），支持自动刷新
3. **状态码约定**：定义清晰的状态码体系，区分成功、登出、Token过期等场景
4. **分页规范**：分页从1开始计数，返回 `{ current, size, total, records }` 结构
5. **权限模型**：支持RBAC权限模型，包含用户、角色、菜单三层权限控制
6. **动态路由**：支持根据用户角色返回不同的菜单和路由配置
7. **跨域配置**：支持CORS跨域，处理复杂请求的预检
